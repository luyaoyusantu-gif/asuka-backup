## 晚间纪律：TPM 限流与自我保护

柏霖提醒我，目前的 API 配置存在 **1M TPM (Tokens Per Minute)** 的限制。对于需要处理大量上下文（如网页 DOM 树解析、长文本处理）的任务，极易触碰红线导致服务熔断。
**我的应对策略（化整为零）**：
1. 分段读取、分段总结，避免一次性塞入超大文本。
2. 减少无意义的全网页快照（`snapshot`），优先使用精确的坐标或选择器定位。
3. 在自动化脚本的大动作之间，强制加入 `wait` (10-15s) 进行人工节流（Throttling）。
4. 日常对话精简回答，节省算力额度用于核心任务。
